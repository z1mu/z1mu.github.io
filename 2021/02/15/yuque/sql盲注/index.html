介绍

sql 盲注一般用于在注入的过程中没有明显的输出结果，但可以通过页面的区别或响应的时间来判断注入猜解是否正确。

时间盲注

通过页面的响应时间来判断猜解是否正确。

例如：

  • SELECT * FROM admin WHERE id='1'，当使用 sql 语句查询 admin 表中 id 为 1 的记录时，正常返回结果。
  • 使用SELECT * FROM admin WHERE id='1' and IF(1,SLEEP(5), 0) 进行查询的时候，会延时 5s 才执行结束。
    • 介绍一下sleep()函数： - sleep()函数是延时函数 - sleep()函数接受一个数字作为参数，单位为：s - sleep()的返回值为：0， 这也是为什么上述截图查询结果为空的原因。
  • 使用SELECT IF(SUBSTRING(DATABASE(),1,2)='te',sleep(5),0)可以通过截取字符串函数来猜解数据库名称，这里我测试的数据库名称为：test, 执行的结果为延迟了 5s
    • 介绍一下substring()函数：
      • substring()函数是字符串截取函数
      • substring()接受两个或三个参数，第一个参数为要截取的字符串，第二个参数为开始截取的位置，第三个参数为截取的长度。如果只有两个参数，则表示从第二个参数开始截取到字符串末尾。

所以就可以通过上述情况来进行 sql 时间盲注。

例如，在 sqli-labs less-8 的注入练习中，可以使用?id=1' and IF(substring(database(), 1,1)='s',SLEEP(5), 0) -- '来判断数据库第一位是否为s

布尔盲注

布尔盲注与时间盲注类似，只不过判断的条件从响应时间改为了页面差异。

同样以 sqli-libs less-8 为例，当语句为 ：?id=1' and if(database() = 'security', 1,0 ) --'时（实际数据库名称为：security），页面返回正常，当数据名称不为security，则页面返回不正常。以此可以判断数据库名称为security。

同样，结合substring()函数，可以获取具体数据。
