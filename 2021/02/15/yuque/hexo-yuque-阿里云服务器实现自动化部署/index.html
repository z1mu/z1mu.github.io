hexo-yuque-阿里云服务器实现自动化部署

发表于 2021-02-15 更新于 2021-03-02

通过使用 hexo 做博客前端，语雀做后端，实现语雀发布文章，自动同步到阿里云服务器上。

HEXO 官网：https://hexo.io/zh-cn/。 Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

语雀 官网：https://www.yuque.com/ 。「语雀」是一个「专业的云端知识库」，孵化自 蚂蚁集团 ，是 体验科技 理念下的一款创新产品，已是 10 万阿里员工进行文档编写、知识沉淀的标配。

安装博客框架

初始化博客框架

在 GitHub 上新建一个仓库

创建一个名为 blog （可自定义）的私有仓库， 并克隆到本地。**将使用 hexo 创建的 blog_temp 中的内容全部复制到克隆下来的 blog 文件夹中。迁移完成后，即可删除 blog_temp 文件夹。

我们可以先在本地调好样式，然后再同步到 GitHub。

本地启动 hexo，预览样式

清除缓存： hexo clean
生成文件： hexo generate
启动服务： hexo server
或者组成一条命令： hexo clean && hexo g && hexo s

访问http://localhost:4000即可查看生成的博客。

更换 hexo 主题

这里我选用的是 next 主题， 更多主题样式可以参考：https://hexo.io/themes/

next 主题 GitHub：https://github.com/theme-next/hexo-theme-next

克隆 next 主题到本地 blog 目录下。

这里克隆完成后，建议删除 themes/next/ 目录下的 .git 、 .github 目录，这两个目录默认是隐藏的，可以百度搜索查看隐藏文件的教程。

打开根目录下的 _config.yml 文件，找到 theme: ，将值改为: next

需要注意的是，博客根目录和主题根目录都有 _config.yml 文件，修改的时候需要注意是哪个文件。

修改完毕后，重新启动服务，查看效果。
